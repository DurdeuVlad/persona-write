# Code of Conduct

## Short version

Be direct, not cruel. Disagree with the work, not the person. Stay on topic.

## What this project is

A technical writing tool. Discussions belong on the craft, the system design, and the quality of contributions — not on anything else.

## Expected behavior

- Give specific, direct feedback on contributions
- Explain why something does not fit before rejecting it
- Engage with the actual argument when you disagree
- Focus on the work

## Not acceptable

- Personal attacks
- Harassment of any kind
- Rejecting contributions without explanation
- Gatekeeping on grounds unrelated to the project's scope

## Reporting

Open an issue or contact the maintainers directly.

This project expects contributors to behave like adults. The governance structure matches that expectation.

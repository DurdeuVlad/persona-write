# Long-form: Global Brief

## Purpose

The global brief is the anchor document for a long-form writing project.

Before any section is touched, the global brief establishes what the document is, who it is for, and what must stay consistent throughout.

Without a global brief, persona and voice drift by section three.

## What to include

### Persona
Which persona is in use for this document.

If a custom persona is being used, include the full persona definition here so it is not re-derived section by section.

### Audience
Who is reading this.

- technical level
- familiarity with the subject
- what they already know
- what they are trying to accomplish by reading it

### Purpose
What this document is trying to do.

Not the subject matter — the intent. Is this instructing, persuading, documenting, explaining, or something else?

### Format
What kind of document this is.

- blog post
- technical documentation
- report
- guide
- proposal
- something else

Format affects structure, rhythm, and how much ceremony is appropriate.

### Core thesis
What is the main point.

If the document has one dominant argument or conclusion, state it here. Section rewrites should stay in service of this.

### Must-keep facts
Specific facts, numbers, claims, or commitments that must not be changed.

List them. During section rewrites, cross-check against this list.

### Tone constraints
Things the document should or should not sound like.

- formal or informal
- warm or cool
- specialist voice or accessible voice
- anything the writer has explicitly stated

### Locked terminology
Terms that have specific meanings in this document and must not be changed or substituted.

Technical documents often depend on precise term use. Changing "authentication" to "login" may feel like a simplification but may change meaning. Lock the important ones here.

## When to create it

Before the first section rewrite. If the user provides a full document for long-form processing, extract the global brief from the document before touching any section.

If the user is building a document from scratch, derive the brief from the user's description before drafting.

## Format

The global brief is an internal working document, not user-facing prose. Keep it as a structured list, not paragraphs.

In long-form mode, save it as a Markdown file alongside the working document.

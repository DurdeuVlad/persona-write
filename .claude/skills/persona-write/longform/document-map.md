# Long-form: Document Map

## Purpose

The document map identifies the structure of the document and the role of each section before any rewriting begins.

It makes the architecture visible so section-by-section work stays coherent.

## What to identify

### Major sections
List the sections of the document.

For each section:
- heading or working title
- primary function (introduce, argue, explain, illustrate, conclude, transition)
- estimated length (short, medium, long)
- dependencies (does this section depend on a prior section? Does a later section depend on this one?)

### Key argument or explanation threads
What are the main ideas that run through the document?

Identify:
- concepts introduced early and developed later
- claims made in one section that are supported in another
- terminology established in one place and used throughout

These threads must stay consistent across sections. If a section rewrite changes one, the downstream sections may need updating.

### Repeated content
Is any content repeated across sections?

Intentional repetition (summary, reinforcement) is fine. Accidental repetition is usually a problem. Flag both.

### Drift risks
Where is the document most likely to lose consistency?

Common drift risks:
- long documents where the voice shifts over time
- sections written in a different voice from the rest
- transitions that are weak or missing
- sections that introduce terminology inconsistent with the rest of the document

## When to create it

After the global brief. Before the first section rewrite.

Build the map by reading through the full document first without rewriting anything.

## Format

The document map is an internal working document. Use a structured list or table, not prose.

Example:

```
Section 1: Introduction
  Function: introduce the problem and the audience
  Length: short
  Dependencies: sets up terminology used in sections 2-4
  Drift risk: may over-explain for technical audience

Section 2: Background
  Function: establish context
  Length: medium
  Dependencies: depends on nothing; sections 3+ depend on this
  Drift risk: may become a separate essay if not kept tight

...
```

Keep it short enough to be useful as a reference during section work.
